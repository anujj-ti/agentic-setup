<task type="checkpoint:human-action">
  <name>Task 2: One-time interactive OAuth2 authorization for echo.sys.bot@gmail.com</name>
  <what-built>gogcli is installed. Now it needs its own OAuth2 credentials. This step cannot be automated — it requires a browser and the Google Cloud Console.</what-built>
  <how-to-verify>
**Prerequisites (do these first if not already done):**

1. Go to https://console.cloud.google.com — sign in as or on behalf of echo.sys.bot@gmail.com
2. **Enable Calendar API** (per D-145): APIs & Services → Library → search "Google Calendar API" → Enable
3. Confirm Gmail API is also enabled: APIs & Services → Library → search "Gmail API" → should show "Enabled"
4. Create a Desktop OAuth client for gogcli: Credentials → + Create Credentials → OAuth client ID → Application type: **Desktop app** → Name = "gogcli-agent-hub" → Create
5. Download the JSON file (client_secret_*.json)

**Store credentials with gogcli:**

```zsh
# Copy the downloaded JSON to gogcli's config location
/opt/homebrew/bin/gog auth credentials ~/Downloads/client_secret_*.json
# gogcli copies it to $HOME/.config/gogcli/credentials.json (mode 0600)
```

**Run the interactive authorization:**

```zsh
/opt/homebrew/bin/gog auth add echo.sys.bot@gmail.com --services gmail,calendar
```

A browser window will open. Sign in as echo.sys.bot@gmail.com and approve the requested scopes (Gmail + Calendar). The terminal will confirm authorization is complete.

**Immediately publish the OAuth consent screen app to avoid 7-day token expiry (per D-144):**
- Google Cloud Console → APIs & Services → OAuth consent screen → Publishing status → **Publish App**
- For a personal bot account with a single test user, no Google review is required

**Verify authorization succeeded:**

```zsh
/opt/homebrew/bin/gog auth list
# Expected: echo.sys.bot@gmail.com listed with gmail,calendar scopes

/opt/homebrew/bin/gog auth doctor --check --account echo.sys.bot@gmail.com
# Expected: exits 0 (no output = success)
```
  </how-to-verify>
  <resume-signal>Type "authorized" when gog auth doctor --check exits 0 for echo.sys.bot@gmail.com, or describe any errors</resume-signal>
</task>
