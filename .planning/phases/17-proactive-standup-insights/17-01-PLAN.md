Create scripts/standup-insights.sh — a pure jq + zsh script that reads the
standup-brief.sh JSON output on stdin and emits an enhanced JSON document
containing classified items, a ranked tackle-first list, and pattern alerts.

On Track) without any LLM call in the classification path. This is the engine
that turns a raw activity dump into a decision-support surface.

{
  "ok": true,
  "data": {
    "insights": {
      "classified_items": [...],
      "tackle_first": [...],
      "patterns": [...]
    }
  }

Scripts lib pattern (scripts/lib/json-response.sh):
  json_ok  '{"key":"value"}'   → emits {"ok":true,"data":{...}} to stdout, exits 0
  json_fail "code" "message"   → emits {"ok":false,"error":"..."} to stdout, exits 1

Classification decisions (D-401 through D-413):
  D-401 Blocked:  ci_failures item where length of ci_failures > 0 AND updatedAt/createdAt
                  has not been updated in last 2h (createdAt is older than now-2h)
  D-402 At Risk:  stale_prs item where updatedAt > 24h ago AND reviewRequests present
                  (reviewDecision == "CHANGES_REQUESTED" is also At Risk)
  D-403 On Track: everything else with recent activity
  D-404 Classification in standup-insights.sh — no LLM call
  D-405 Tackle-first rank: (1) Blocked, (2) At Risk, (3) P0 label / CI failure, (4) most-recently-updated
  D-406 Each tackle_first item: title, status, source_field, reason
  D-407 tackle_first always present ([] when empty)
  D-408 Pattern fires when 3+ items share same signal type
  D-409 Pattern shape: {"type":"ci_failures","count":4,"label":"4 CI failures overnight — possible systemic issue"}
  D-412 Output shape: insights.tackle_first, insights.patterns, insights.classified_items
  D-413 Label-only mode: cite source_field, no free-form interpretation

  <action>
Create scripts/standup-insights.sh with shebang #!/usr/bin/env zsh and set -euo pipefail.

Source scripts/lib/json-response.sh for json_ok and json_fail.

Explicit binary paths (per CLAUDE.md — never rely on PATH):
  JQ=/opt/homebrew/bin/jq

Read all stdin into a variable RAW_INPUT. If stdin is empty, call json_fail "no-input" "No standup JSON on stdin" and exit 1.

Parse RAW_INPUT with $JQ: extract .data.ci_failures (default []), .data.stale_prs (default []), .data.merged_prs (default []).

If $JQ exits non-zero (malformed JSON), call json_fail "invalid-json" "Could not parse standup JSON from stdin".

Compute NOW_EPOCH using date -u '+%s' (BSD date, macOS).
Compute TWO_H_AGO_EPOCH as NOW_EPOCH - 7200.
Compute TWENTY_FOUR_H_AGO as an ISO8601 string: date -u -v-24H '+%Y-%m-%dT%H:%M:%SZ'.

Classification — ci_failures (D-401 Blocked):
  For each element in ci_failures array, compute its createdAt epoch via $JQ with @sh and
  date -j -f '%Y-%m-%dT%H:%M:%SZ' ... '+%s' (BSD date parse).
  If createdAt_epoch < TWO_H_AGO_EPOCH → Blocked.
  source_field: "ci_failures[N]" where N is the zero-based index.
  reason: "CI failure on branch [headBranch] — not updated in over 2h"

Classification — stale_prs (D-402 At Risk):
  For each element in stale_prs, parse updatedAt epoch.
  If updatedAt_epoch < (NOW_EPOCH - 86400) → At Risk (updatedAt older than 24h ago).
  source_field: "stale_prs[N]"
  reason: "PR #[number] has [reviewDecision] and has not been updated in over 24h"

Classification — merged_prs (D-403 On Track):
  All merged_prs items are On Track — they are completed work.
  source_field: "merged_prs[N]"
  reason: "Merged overnight — no action needed"

All unclassified ci_failures (createdAt within 2h) and stale_prs (updatedAt within 24h) →
  On Track.
  source_field: "ci_failures[N]" or "stale_prs[N]"
  reason: "Recent activity — monitoring"

Build classified_items JSON array using $JQ -n with --argjson inputs.

Build tackle_first: filter classified_items for Blocked + At Risk, sort by:
  1. status == "Blocked" first (rank 0)
  2. status == "At Risk" (rank 1)
  3. On Track items with source from ci_failures (rank 2)
  4. On Track items by recency (rank 3)
Limit to 5 items. Use $JQ with sort_by and limit.
Emit tackle_first as [] when no qualifying items exist (D-407).

Build patterns: group classified_items by signal type using $JQ group_by(.source_field | split("[")[0]).
For each group with length >= 3, emit:
  {"type": "<signal_type>", "count": N, "label": "N <signal_type> overnight — possible systemic issue"}
where <signal_type> is "ci_failures" or "stale_prs" (derived from source_field prefix).
Emit patterns as [] when no group reaches 3 (D-410 — no trend claims without 3+ days history).

Assemble final output with json_ok:
  {"insights":{"classified_items":$CLASSIFIED,"tackle_first":$TACKLE,"patterns":$PATTERNS}}

Verify the script is executable (chmod +x). Output goes to stdout only; logs go to stderr.
