From Task Orchestrator SOUL.md — Beads epic creation pattern (use exact bd syntax):
```zsh
BEADS_DIR="$HOME/.openclaw/beads"
BD="/opt/homebrew/opt/node@24/bin/bd"

EPIC=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Implement issue #N: <title> (OWNER/REPO)" \
  -t epic -p 1 --json | jq -r '.id')

T1=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Design proposal for #N: <title>" \
  --parent "$EPIC" --json | jq -r '.id')
T2=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Implementation for #N: <title>" \
  --parent "$EPIC" --deps "$T1" --json | jq -r '.id')
T3=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Self-review for #N: <title>" \
  --parent "$EPIC" --deps "$T2" --json | jq -r '.id')
T4=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Quality-review evidence for #N: <title>" \
  --parent "$EPIC" --deps "$T3" --json | jq -r '.id')
T5=$(BEADS_DIR="$BEADS_DIR" "$BD" create "Open PR for #N: <title>" \
  --parent "$EPIC" --deps "$T4" --json | jq -r '.id')
```

Create `devbot-execute-cycle.sh`. This script handles one Beads subtask execution cycle. DevBot calls it once per task after claiming from `bd ready`. The script's behavior varies based on `TASK_TYPE` — it performs the work appropriate to that subtask phase and closes the task with a factual evidence string.

Requirements:
- Shebang: `#!/usr/bin/env zsh` + `set -euo pipefail`
- Positional args: `TASK_ID="${1:?task id required}"`, `TASK_TYPE="${2:?task type required}"`, `REPO="${3:?repo required}"`, `ISSUE_NUM="${4:?issue number required}"`
- Constants: `BEADS_DIR="$HOME/.openclaw/beads"`, `BD="/opt/homebrew/opt/node@24/bin/bd"`, `GH="/opt/homebrew/bin/gh"`
- Step 1 — Claim the task:
  `BEADS_DIR="$BEADS_DIR" "$BD" update "$TASK_ID" --claim --json >&2`
  Log to stderr; failure exits with error JSON
- Step 2 — Execute based on TASK_TYPE:

  `design`: Write a design note to stderr. In Phase 8 this is a scaffold — log "DevBot design scaffold: actual design generation is Phase 12 territory." Close with evidence: `"Design scaffold created: placeholder for Phase 12 AI design generation. Issue #${ISSUE_NUM} in ${REPO}."`

  `implement`: Write implementation note to stderr. In Phase 8 this is a scaffold — log "DevBot implement scaffold: actual code generation is Phase 12 territory." Close with evidence: `"Implementation scaffold: actual code generation deferred to Phase 12. Issue #${ISSUE_NUM} in ${REPO}."`

  `self-review`: Log self-review scaffold. Close with evidence: `"Self-review scaffold: placeholder checklist — functionality, tests, style. Phase 12 will perform actual review. Issue #${ISSUE_NUM}."`

  `qa-evidence`: Log QA evidence scaffold. Close with evidence: `"QA evidence scaffold: placeholder — test run results, lint output. Phase 12 integration required. Issue #${ISSUE_NUM}."`

  `open-pr`: This is NOT a scaffold. T5 DOES create a real draft PR using:
    - Branch name: `devbot/issue-${ISSUE_NUM}` (check if it exists first with `git show-ref --verify --quiet refs/heads/devbot/issue-${ISSUE_NUM}`)
    - If branch does not exist: `"$GH" repo clone "$REPO" /tmp/devbot-pr-${ISSUE_NUM} --quiet 2>/dev/null; cd /tmp/devbot-pr-${ISSUE_NUM}; git checkout -b "devbot/issue-${ISSUE_NUM}"`
    - Create the PR: `"$GH" pr create -R "$REPO" --base main --head "devbot/issue-${ISSUE_NUM}" --title "feat: issue #${ISSUE_NUM} (closes #${ISSUE_NUM})" --body "Implements #${ISSUE_NUM}. Autonomous dev scaffold (Phase 8) — implementation in Phase 12." --draft`
    - Capture the PR URL from stdout
    - Close with evidence: `"Draft PR opened: <PR_URL>. Branch: devbot/issue-${ISSUE_NUM}. No code changes in Phase 8 scaffold."`

  Unknown TASK_TYPE: `{"ok": false, "error": "unknown task type: $TASK_TYPE"}` exit 1

- Step 3 — Close the task with evidence from step 2:
  `BEADS_DIR="$BEADS_DIR" "$BD" close "$TASK_ID" --reason "$CLOSE_REASON" --json`

- stdout (final line): `{"ok": true, "task_id": "$TASK_ID", "task_type": "$TASK_TYPE", "closed": true}`
- Make executable: `chmod +x scripts/devbot-execute-cycle.sh`

CRITICAL: T5 opens a draft PR only. The script must NOT call `gh pr merge` under any circumstances in Phase 8. Add a comment: `# Phase 8: open draft PR only. Merge gate is Phase 10 (Notion pre-log required).`
