Smoke test source (RESEARCH.md "Validation Architecture → Phase Requirements → Test Map" and 01-PATTERNS.md "scripts/infra-verify.sh"):
- check() helper tracks PASS/FAIL/FAILURES array
- Each check runs a single assertion command, increments counters
- Final JSON: {"ok":true,"data":{"passed":N,"failed":0}} on green, {"ok":false,"error":"smoke_tests_failed","data":{...}} on red

Grep gate hygiene: when checking script files for required strings, use `grep -v '^#' | grep -q ...` so comments don't satisfy the assertion.

Script invocation convention (CLAUDE.md): all repo scripts are `#!/usr/bin/env zsh` and MUST be invoked with `zsh` (never `bash`). Calling `bash scripts/foo.sh` runs zsh-syntax code through bash 3.2 and breaks at `print` builtins.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo `.openclaw/` → runtime `~/.openclaw/` | Stow symlinks make repo files the runtime source of truth — the gateway daemon reads files whose content lives in a git-tracked location |
| launchd → ~/.openclaw/scripts/openclaw-secrets.sh | Future secret vars will be sourced from this symlinked file into the daemon's environment |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | jobs.json conflict masks real config corruption | mitigate | `rm -f ~/.openclaw/cron/jobs.json` is targeted (single file, known gateway-owned). The marker comment (D-11) ensures additional conflicts are explicitly added rather than silently swallowed via `--adopt` or `--override`. |
| T-04-02 | Tampering | Pre-existing plain ~/.openclaw/openclaw.json overwrite | mitigate | Human checkpoint (Task 2) backs the file up to openclaw.json.pre-stow before deletion. No automated overwrite. |
| T-04-03 | Information Disclosure | Permissions of ~/.openclaw after stow | mitigate | Stow preserves source permissions. Stub files in this plan contain no secrets. RESEARCH.md notes `chmod 700 ~/.openclaw` is flagged by `openclaw doctor`; Plan 01-05 verifies via /openclaw-status. |
| T-04-04 | Denial of Service | stow conflict halts deploy on every gateway restart | mitigate | The deploy script's first action is removing jobs.json. Pitfall 1 is structurally prevented for the documented case; the marker comment lets future conflicts be added explicitly. |
| T-04-05 | Elevation of Privilege | Stow targeting an arbitrary directory via $REPO_DIR override | accept | REPO_DIR default is hardcoded to $HOME/Documents/agentic-setup. Operators running the script with an unusual REPO_DIR override are explicitly doing so. No external input. |
</threat_model>
