json-response.sh pattern (write this exact shape):
```zsh
#!/usr/bin/env zsh
# lib/json-response.sh — cc-openclaw output convention
# Source this file; call json_ok '{"key":"val"}' or json_err "message"

json_ok() {
  local payload="${1:-{}}"
  printf '{"ok":true,"data":%s}\n' "$payload"
}

json_err() {
  local message="${1:-unknown error}"
  printf '{"ok":false,"error":"%s"}\n' "$message" >&2
  printf '{"ok":false,"error":"%s"}\n' "$message"
  exit 1
}
```

gh issue create return value:
- stdout: https://github.com/OWNER/REPO/issues/42
- Extract issue number: `basename "$ISSUE_URL"` → 42

RESEARCH.md Pattern 2 (verified on gh 2.69.0):
```zsh
GH_ARGS=(issue create --repo "$REPO" --title "$TITLE" --body "$BODY")
[[ -n "${LABEL:-}" ]]     && GH_ARGS+=(--label "$LABEL")
[[ -n "${MILESTONE:-}" ]] && GH_ARGS+=(--milestone "$MILESTONE")
[[ -n "${PROJECT:-}" ]]   && GH_ARGS+=(--project "$PROJECT")
[[ -n "${ASSIGNEE:-}" ]]  && GH_ARGS+=(--assignee "$ASSIGNEE")
ISSUE_URL=$($GH "${GH_ARGS[@]}" 2>/dev/null)
```
