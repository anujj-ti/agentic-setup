<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Human Telegram account → bot | Pairing flow gates who can DM the bot; only paired users can send messages |
| `openclaw pairing approve` CLI | Runs as the gateway owner; approves inbound pairing requests; only owner can run this |
| `openclaw message send` CLI | Sends outbound messages on behalf of the bot; requires gateway ownership |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Elevation of Privilege | Telegram pairing | mitigate | `dmPolicy: "pairing"` enforced by config. Only DMs from approved users are processed. Pairing requires owner CLI approval. |
| T-02-08 | Information Disclosure | Pairing code in Telegram chat | accept | Pairing code is low-value: single-use, 1-hour expiry, requires owner terminal access to approve. No secret material disclosed. |
| T-02-09 | Spoofing | `openclaw message send --target` | accept | Only the gateway owner can send outbound messages. Target is owner's own Telegram user ID for verification purposes. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Plan 02-02 installs no packages. Verification only. |
</threat_model>
