<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell stdout → log file | Any command that prints the token value to stdout risks it appearing in terminal scrollback or shell history |
| Keychain → env var → gateway process | Token leaves Keychain only as an env var value — never as a file write or CLI argument |
| openclaw.json → git history | Any literal token in openclaw.json would be committed and visible in perpetuity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | openclaw.json | mitigate | Token stored as `${OPENCLAW_TELEGRAM_MAIN_BOT_TOKEN}` — never literal. Post-write grep gate: `grep -qvE '[0-9]{10}:AA' .openclaw/openclaw.json`. |
| T-02-02 | Information Disclosure | ~/.openclaw/openclaw.json.pre-stow | mitigate | File shredded in Task 2 Step 7, only after gateway confirms token resolves correctly. Shred verified with `ls ... | grep 'No such file'`. |
| T-02-03 | Information Disclosure | git history | mitigate | Pre-commit check: `git diff HEAD~1 \| grep -vE '^(---|\+\+\+\|@@)' \| grep -qvE '[0-9]{10}:AA'`. Token never written to tracked files. |
| T-02-04 | Information Disclosure | shell command history | mitigate | Token transferred via `security ... \| security ...` pipe or stdin-prompt — token value never appears as a CLI argument or in zsh history. |
| T-02-05 | Elevation of Privilege | Telegram dmPolicy | mitigate | `dmPolicy: "pairing"` enforced in config — every new sender requires owner approval. `dmPolicy: "open"` is explicitly forbidden. |
| T-02-06 | Spoofing | Duplicate Telegram polling | mitigate | Token placed ONLY in `channels.telegram.accounts.main.botToken`, never at the top-level `channels.telegram` block. Post-write check: `grep -c 'botToken' ~/.openclaw/openclaw.json` must equal 1. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Phase 2 installs no new packages. Package Legitimacy Audit in RESEARCH.md confirms zero new installs. |
</threat_model>
