Create propose-experiment.js. This script accepts CLI flags: --title, --hypothesis, --method, --successCriteria. All four are required (plus --started defaults to now if not provided). Validation:
  - If any required arg is missing: print {"ok":false,"error":"missing required field: <field>"} to stdout, exit 1
  - hypothesis validation: must be at least 20 characters (basic specificity check — very short hypotheses are likely vague)
  - successCriteria validation: must contain at least one measurable word (number, %, "within", "count", "rate", "zero", "all") — if none: print {"ok":false,"error":"success criteria must contain a measurable outcome"}, exit 1
  - On all validations passed: print {"ok":true,"validated":true,"title":"<title>","hypothesis":"<hypothesis>","method":"<method>","successCriteria":"<criteria>","started":"<ISO8601>"} to stdout, exit 0

Create create-experiment-page.js. This script accepts CLI flags: --title, --hypothesis, --method, --successCriteria, --started (defaults to now). It:
  1. Checks for OPENCLAW_NOTION_TOKEN and OPENCLAW_NOTION_EXPERIMENTS_DB_ID in process.env. If either is missing: stderr "env var not set", stdout {"ok":false,"error":"<var> not set — OPENCLAW_NOTION_EXPERIMENTS_DB_ID must be provisioned in Keychain (security add-generic-password -s openclaw.notion-experiments-db-id)"}, exit 1.
  2. Creates @notionhq/client with auth: process.env.OPENCLAW_NOTION_TOKEN and notionVersion: "2026-03-11"
  3. Calls notion.pages.create() with parent: { database_id: process.env.OPENCLAW_NOTION_EXPERIMENTS_DB_ID }
     Properties:
       Title: title (title array)
       Status: select, name "Draft"
       Hypothesis: rich_text
       Method: rich_text
       Success Criteria: rich_text
       Started: date, start: ISO8601 string
  4. On success: prints ONLY the page ID as a bare string to stdout (same pattern as notion-log-decision.js in Phase 10). Page ID is the primary deliverable — the caller embeds it in the Beads epic metadata.
  5. On any error: stderr for details, stdout {"ok":false,"error":"<msg>"}, exit 1.
  </action>
