Read the current scripts/email-triage.sh first to understand its structure. Then add three behaviors after the main gog gmail search call succeeds and results are captured:

**Addition 1: Skip already-processed IDs (D-162)**

After parsing the thread list from gog output, filter out any thread whose `id` field appears in the processed-ids.jsonl from the agent's memory. The script receives the skip-set as either an environment variable or reads the JSONL directly.

Implementation approach: The script should read `/Users/trilogy/.openclaw/agents/email-triage/memory/processed-ids.jsonl` directly using jq:

```zsh
PROCESSED_IDS_FILE="/Users/trilogy/.openclaw/agents/email-triage/memory/processed-ids.jsonl"
# Build skip set: space-separated list of already-processed IDs
if [[ -f "$PROCESSED_IDS_FILE" ]]; then
  SKIP_IDS=$(jq -r '.id' "$PROCESSED_IDS_FILE" 2>/dev/null | tr '\n' ' ' || echo "")
else
  SKIP_IDS=""
fi
```

After filtering, output only threads whose IDs are not in SKIP_IDS.

**Addition 2: Append new IDs to processed-ids.jsonl (D-162)**

After triage output is produced (successful run only — `ok: true`), append each processed messageId as a JSONL entry:

```zsh
PROCESSED_AT=$(date -u +"%Y-%m-%dT%H:%M:%S.000Z")
for ID in $NEWLY_PROCESSED_IDS; do
  echo "{\"id\":\"${ID}\",\"processedAt\":\"${PROCESSED_AT}\"}" >> "$PROCESSED_IDS_FILE"
done
```

**Addition 3: Trim processed-ids.jsonl to last 500 entries (D-163)**

After appending, trim the file to last 500 lines:

```zsh
ENTRY_COUNT=$(wc -l < "$PROCESSED_IDS_FILE" 2>/dev/null || echo 0)
if [[ "$ENTRY_COUNT" -gt 500 ]]; then
  TMPFILE=$(mktemp)
  tail -500 "$PROCESSED_IDS_FILE" > "$TMPFILE"
  mv "$TMPFILE" "$PROCESSED_IDS_FILE"
fi
```

**Addition 4: gog gmail mark-read after triage completes (D-161)**

After all processed IDs are appended, call mark-read for the processed messages. Use the `--query` form since we already know the messages were `is:unread` when fetched:

```zsh
/opt/homebrew/bin/gog gmail mark-read \
  --account "${OPENCLAW_GMAIL_ACCOUNT:-echo.sys.bot@gmail.com}" \
  --query "is:unread newer_than:1d" \
  --no-input --non-interactive 2>>"$LOG_STDERR" || \
  echo "[warn] mark-read failed — processed-ids.jsonl guard will catch duplicates next run" >&2
```

Note: mark-read failure is non-fatal — the processed-ids.jsonl secondary guard (D-162) handles the next run. Log to stderr only, do not alter the JSON output to stdout.

All existing strict mode (`set -euo pipefail`) and JSON response patterns from the current script must be preserved. Additions go before the final json_ok call.
