First, create .openclaw/agents/devbot/state/.gitkeep as an empty file so the state/ directory is tracked in git.

Then create .openclaw/agents/devbot/scripts/devbot-issue-monitor.sh with the following structure:

Shebang: #!/usr/bin/env zsh
Strict mode: set -euo pipefail
Source: source "$(dirname "$0")/lib/json-response.sh"

Constants block:
  AGENT_DIR="$HOME/.openclaw/agents/devbot"
  STATE_DIR="$AGENT_DIR/state"
  PENDING_DIR="$STATE_DIR/pending-issues"
  LOGS_DIR="$AGENT_DIR/logs"
  TIMESTAMP_FILE="$STATE_DIR/last-issue-timestamp"
  GH="/opt/homebrew/bin/gh"
  REPO="${1:-anujj-ti/agentic-setup}"
  LOG_DATE=$(date '+%Y-%m-%d')
  LOG_FILE="$LOGS_DIR/issue-monitor-${LOG_DATE}.log"

Ensure runtime dirs exist (mkdir -p):
  mkdir -p "$STATE_DIR" "$PENDING_DIR" "$LOGS_DIR"

Load GH_TOKEN from Keychain:
  export GH_TOKEN=$(security find-generic-password -s 'openclaw.github-bot-token' -a 'trilogy' -w 2>/dev/null)

Read last-seen timestamp. If TIMESTAMP_FILE does not exist, default to epoch "1970-01-01T00:00:00Z":
  LAST_SEEN=$( [[ -f "$TIMESTAMP_FILE" ]] && cat "$TIMESTAMP_FILE" || print "1970-01-01T00:00:00Z" )

Log startup to stderr (which cron redirects to LOG_FILE):
  print "[$(date -u '+%Y-%m-%dT%H:%M:%SZ')] issue-monitor: polling $REPO (last_seen=$LAST_SEEN)" >&2

Poll for candidate issues — single gh call returning number, title, labels, createdAt, assignees:
  ISSUES=$("$GH" issue list \
    --repo "$REPO" \
    --label "automation:safe" \
    --assignee "@none" \
    --state open \
    --json number,title,labels,createdAt,assignees \
    --limit 20 \
    2>/dev/null)

Filter to issues newer than LAST_SEEN and NOT having automation:hold label:
  Use jq to filter:
    jq --arg since "$LAST_SEEN" '[
      .[] | select(
        .createdAt > $since and
        ([.labels[].name] | map(select(. == "automation:hold")) | length) == 0
      )
    ] | sort_by(.createdAt)'

Store filtered list in NEW_ISSUES variable.

Count new issues:
  COUNT=$(print "$NEW_ISSUES" | jq 'length')

If COUNT == 0: log "no new issues" to stderr, output json_ok '{"picked_up":0}', exit 0.

Process issues. For each issue (iterate with jq -c '.[]'):
  Extract ISSUE_NUM and ISSUE_TITLE from jq fields.

  Log to stderr: "processing issue #$ISSUE_NUM: $ISSUE_TITLE"

  STEP 1 — Notion pre-log (per SOUL.md mandatory requirement):
    Call notion-log-decision.js before any GitHub mutation:
      PRE_LOG=$(node "$AGENT_DIR/scripts/notion-log-decision.js" \
        "Autonomous pickup of issue #${ISSUE_NUM}" \
        "Issue labeled automation:safe, no assignee, no automation:hold — matches pickup criteria" \
        "gh issue list confirmed: #{$ISSUE_NUM} $ISSUE_TITLE" \
        "reversible: unassign + remove status:in-progress label to cancel" \
        2>/dev/null) || true
    Extract PAGE_ID from PRE_LOG using python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('pageId',''))"
    Log PAGE_ID to stderr. If empty, log warning but continue (Notion failure is not a blocker for pickup).

  STEP 2 — Claim the issue (per D-204):
    "$GH" issue edit "$ISSUE_NUM" \
      --repo "$REPO" \
      --add-assignee "echosysbot" \
      --add-label "status:in-progress" \
      2>&1 >&2

  STEP 3 — Create branch (per D-207):
    "$GH" issue develop "$ISSUE_NUM" \
      --repo "$REPO" \
      --checkout \
      2>&1 >&2
    Capture branch name from gh issue develop output or derive it:
      BRANCH=$("$GH" issue develop "$ISSUE_NUM" --repo "$REPO" --list 2>/dev/null | head -1 | awk '{print $1}')
      If BRANCH is empty, set BRANCH="${ISSUE_NUM}-$(print "$ISSUE_TITLE" | tr '[:upper:]' '[:lower:]' | tr ' ' '-' | tr -cd 'a-z0-9-' | cut -c1-40)"

  STEP 4 — Write issue JSON to pending-issues/ (per D-203):
    print "$NEW_ISSUES" | jq --argjson num "$ISSUE_NUM" '.[] | select(.number == $num)' \
      > "$PENDING_DIR/${ISSUE_NUM}.json"

  STEP 5 — Spawn DevBot session for the Beads execution cycle (per D-203):
    Note: The cron invocation of this script is not inside an OpenClaw agent session,
    so openclaw sessions spawn cannot be called directly from shell.
    Instead, write the issue number to a queue file:
      print "$ISSUE_NUM" >> "$STATE_DIR/pickup-queue.txt"
    The DevBot agent's AGENTS.md startup check (added in Plan 04) will read this queue
    and process pending issues at the next cron-triggered session.

  STEP 6 — Open draft PR (per D-208 + D-210):
    Check if branch has any commits beyond the base. If gh issue develop created the branch,
    there may be no commits yet. The draft PR creation is best-effort at monitor time;
    devbot-execute-cycle.sh handles the full PR lifecycle after code work.
    For the monitor's purpose: attempt gh pr create and capture URL. If it fails (no commits yet), skip.
      PR_URL=$("$GH" pr create \
        --repo "$REPO" \
        --head "$BRANCH" \
        --base main \
        --fill \
        --body "Resolves #${ISSUE_NUM}" \
        --draft \
        2>/dev/null) || PR_URL=""

  STEP 7 — Enable auto-merge (per D-209) if PR was created:
    If PR_URL is non-empty:
      "$GH" pr merge "$PR_URL" \
        --auto \
        --squash \
        --delete-branch \
        --repo "$REPO" \
        2>&1 >&2 || true

After processing all issues, update LAST_SEEN to the createdAt of the most recent processed issue:
  NEW_LAST_SEEN=$(print "$NEW_ISSUES" | jq -r 'last | .createdAt')
  print "$NEW_LAST_SEEN" > "$TIMESTAMP_FILE"

Output final JSON:
  json_ok "{\"picked_up\":$COUNT,\"repo\":\"$REPO\",\"last_seen\":\"$NEW_LAST_SEEN\"}"

Make executable: chmod +x .openclaw/agents/devbot/scripts/devbot-issue-monitor.sh

IMPORTANT: The iteration over issues must handle the case where jq returns no output gracefully. Use:
  print "$NEW_ISSUES" | jq -c '.[]' | while IFS= read -r issue; do
    ISSUE_NUM=$(print "$issue" | jq -r '.number')
    ISSUE_TITLE=$(print "$issue" | jq -r '.title')
    ...
  done
This zsh while-read loop is safe under set -euo pipefail as long as the inner commands handle their own errors.
