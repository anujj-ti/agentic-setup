<objective>
Create devbot-stale-claim-guard.sh — the hourly cron script (D-205) that finds issues stuck in status:in-progress with no recent branch activity and unassigns them, returning them to the pickup pool.

Purpose: Without this guard, a DevBot crash or error during issue processing would permanently block an issue — it would stay assigned with status:in-progress and never get re-processed. The 2-hour timeout ensures the queue self-heals.
Output: devbot-stale-claim-guard.sh — safe to run hourly via launchd cron (wired in Plan 04).
</objective>

- D-206: automation:hold blocks pickup — the guard does NOT remove automation:hold (that's a user-managed kill switch)

How to detect staleness:
  1. List all open issues with label "status:in-progress" (regardless of assignee — guard covers all bot-claimed issues)
  2. For each: get the linked branch (from gh issue develop --list)
  3. Check last commit time on that branch via gh api
  4. If last commit is older than 2h (or branch doesn't exist), mark as stale
  5. Unclaim: gh issue edit N --remove-assignee echosysbot --remove-label "status:in-progress"
  6. Comment: gh issue comment N --body "echosysbot: timed out, unclaiming"

Create .openclaw/agents/devbot/scripts/devbot-stale-claim-guard.sh with:

Shebang: #!/usr/bin/env zsh
Strict mode: set -euo pipefail
Source: source "$(dirname "$0")/lib/json-response.sh"

Load GH_TOKEN:
  export GH_TOKEN=$(security find-generic-password -s 'openclaw.github-bot-token' -a 'trilogy' -w 2>/dev/null)

Constants:
  GH="/opt/homebrew/bin/gh"
  REPO="${1:-anujj-ti/agentic-setup}"
  STALE_HOURS=2
  CUTOFF_ISO=$(date -u -v"-${STALE_HOURS}H" '+%Y-%m-%dT%H:%M:%SZ')

Log guard start to stderr:
  print "[$(date -u '+%Y-%m-%dT%H:%M:%SZ')] stale-claim-guard: checking $REPO (cutoff=$CUTOFF_ISO)" >&2

List all open issues with status:in-progress label:
  IN_PROGRESS=$("$GH" issue list \
    --repo "$REPO" \
    --label "status:in-progress" \
    --state open \
    --json number,title,assignees,createdAt \
    --limit 50 \
    2>/dev/null)

Count:
  COUNT=$(print "$IN_PROGRESS" | jq 'length')
  print "[$(date -u '+%Y-%m-%dT%H:%M:%SZ')] stale-claim-guard: found $COUNT in-progress issues" >&2

If COUNT == 0: json_ok '{"stale_unclaimed":0}', exit 0.

Process each in-progress issue:
  UNCLAIMED=0
  print "$IN_PROGRESS" | jq -c '.[]' | while IFS= read -r issue; do
    ISSUE_NUM=$(print "$issue" | jq -r '.number')
    ISSUE_TITLE=$(print "$issue" | jq -r '.title')
    print "checking issue #$ISSUE_NUM: $ISSUE_TITLE" >&2

    # Find linked branch via gh issue develop --list
    BRANCH=$("$GH" issue develop "$ISSUE_NUM" --repo "$REPO" --list 2>/dev/null | head -1 | awk '{print $1}' || print "")

    LAST_COMMIT_TIME=""
    if [[ -n "$BRANCH" ]]; then
      # Get last commit time on the branch
      LAST_COMMIT_TIME=$("$GH" api "repos/$REPO/branches/$BRANCH" --jq '.commit.commit.committer.date' 2>/dev/null || print "")
    fi

    # Determine staleness:
    # - Branch doesn't exist → stale (issue was claimed but branch never created)
    # - Branch exists but last commit is older than cutoff → stale
    # - Branch exists and last commit is newer than cutoff → not stale
    IS_STALE=false

    if [[ -z "$BRANCH" || -z "$LAST_COMMIT_TIME" ]]; then
      IS_STALE=true
      print "issue #$ISSUE_NUM stale: no branch or no commits" >&2
    elif [[ "$LAST_COMMIT_TIME" < "$CUTOFF_ISO" ]]; then
      IS_STALE=true
      print "issue #$ISSUE_NUM stale: last commit $LAST_COMMIT_TIME < cutoff $CUTOFF_ISO" >&2
    else
      print "issue #$ISSUE_NUM active: last commit $LAST_COMMIT_TIME" >&2
    fi

    if [[ "$IS_STALE" == "true" ]]; then
      # Unclaim: remove assignee and label
      "$GH" issue edit "$ISSUE_NUM" \
        --repo "$REPO" \
        --remove-assignee "echosysbot" \
        --remove-label "status:in-progress" \
        2>&1 >&2 || true

      # Add comment
      "$GH" issue comment "$ISSUE_NUM" \
        --repo "$REPO" \
        --body "echosysbot: timed out, unclaiming (no branch activity in ${STALE_HOURS}h — issue is available for pickup again)" \
        2>&1 >&2 || true

      print "unclaimed issue #$ISSUE_NUM" >&2
      UNCLAIMED=$((UNCLAIMED + 1))
    fi
  done

Output result:
  json_ok "{\"stale_unclaimed\":$UNCLAIMED,\"repo\":\"$REPO\",\"cutoff\":\"$CUTOFF_ISO\"}"

Make executable: chmod +x .openclaw/agents/devbot/scripts/devbot-stale-claim-guard.sh

Note on UNCLAIMED counter and while loop: the while loop runs in a subshell in zsh under some configurations. To ensure the counter persists, use a temp file to accumulate the count and read it after the loop:
  UNCLAIMED_FILE=$(mktemp)
  trap 'rm -f "$UNCLAIMED_FILE"' EXIT
  print "0" > "$UNCLAIMED_FILE"
  ... in loop: UNCLAIMED=$(( $(cat "$UNCLAIMED_FILE") + 1 )); print "$UNCLAIMED" > "$UNCLAIMED_FILE"
  UNCLAIMED=$(cat "$UNCLAIMED_FILE")
This is the correct zsh pattern for accumulating a counter across a pipeline-driven while loop.
