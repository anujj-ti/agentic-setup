Create `scripts/synapse-query-learnings.sh` following the exact pattern of `scripts/synapse-record-learning.sh`.

Script specification (per D-303, D-304, D-305):

- Shebang: `#!/usr/bin/env zsh`
- Strict mode: `set -euo pipefail`
- Args: `<project_id> <applies_to_tag> [limit]` — limit defaults to 5 when not supplied
- Usage guard: if [[ $# -lt 2 ]] → print usage to stderr, exit 1
- SYNAPSE_TOKEN guard: if [[ -z "${SYNAPSE_TOKEN:-}" ]] → print "synapse-query-learnings: SYNAPSE_TOKEN not set — skipping" to stderr, print empty string to stdout (no output = no bullets = agent sees nothing = non-blocking), exit 0
- SYNAPSE_URL default: `"${SYNAPSE_URL:-https://cnu.synapse-os.ai}"`
- Build JSON body using python3 (same safe-quoting pattern as synapse-record-learning.sh):
  - `applies_to` is a JSON array containing the single tag string
  - `limit` is an integer
- curl call: `/usr/bin/curl -sS -X POST "${SYNAPSE_URL}/v1/intent/synapse.learning.query"` with Authorization Bearer and Content-Type headers, body from python3 output
- On curl failure (|| block): print error to stderr, exit 0 — never block
- Parse response: use python3 to check `ok` field. If `ok` is false or missing: print "# Synapse Learnings: <tag>\n(unavailable)" to stdout, exit 0
- If `ok` is true: parse `data.learnings` array. If empty: print "# Synapse Learnings: <tag>\n- (no learnings found for this domain)" to stdout, exit 0
- If learnings present: print formatted output:
  ```
  # Synapse Learnings: <tag>
  - <claim> [confidence: <confidence>]
  - <claim> [confidence: <confidence>]
  ```
  (one bullet per learning, up to limit)
- Make script executable: `chmod +x scripts/synapse-query-learnings.sh`

The script MUST exit 0 in every error path (token missing, network error, non-ok response, empty learnings). Only exit 1 on bad arg count (developer error, not runtime error).
