  <action>
Create `.openclaw/agents/ci-monitor/DREAM-ROUTINE.md`. CI Monitor is stateless between sessions (state lives in state/last-seen-runs.json) so the dream routine is minimal — its primary purpose is distilling CI failure patterns observed across sessions and merging relevant Synapse cross-silo learnings.

The dream routine should specify:

**Trigger:** Nightly cron at 23:10 Asia/Kolkata (5 minutes after task-orchestrator's 23:05 run to avoid concurrent LLM load).

**Memory location:** `~/.openclaw/agents/ci-monitor/memory/` — the routine should create this if absent. Write a MEMORY.md file here with CI failure pattern summaries.

**Process (in order):**

1. Read `state/last-seen-runs.json` to understand recent CI activity — count failure vs success runs in the last 24h.
2. Read `memory/MEMORY.md` if it exists — existing CI failure pattern context.
3. Query Synapse for cross-silo learnings to merge:
   ```zsh
   SYNAPSE_CI=$(bash ~/Documents/agentic-setup/scripts/synapse-query-learnings.sh \
     project.agentic-setup ci 5 2>/dev/null)
   SYNAPSE_GH=$(bash ~/Documents/agentic-setup/scripts/synapse-query-learnings.sh \
     project.agentic-setup github 3 2>/dev/null)
   ```
4. Distill a CI failure pattern summary — focus only on patterns that have recurred (same repo, same workflow, same failing step).
5. Update `memory/MEMORY.md` with the distillation. Structure:
   - `## CI Failure Patterns` — recurring failures by repo/workflow
   - `## Cross-Silo Learnings` — bullets from Synapse (capped at 500 tokens of new content per cycle per D-311)
   - `## Last Updated` — ISO date
6. Keep MEMORY.md within 1,000 tokens total (CI Monitor is a narrow-domain agent — less historical context needed vs orchestrators).

**Rules:**
1. NEVER include credentials, tokens, or run IDs in MEMORY.md.
2. Synapse content cap: at most 500 tokens of new Synapse bullets per cycle (D-311). If Synapse returns more, include only the top 3 by recency.
3. If state/last-seen-runs.json is empty or shows no failures in 24h: write a one-line MEMORY.md entry "No CI failures in past 24h" and exit.
4. Synapse unavailability is non-blocking — skip the cross-silo section if SYNAPSE_CONTEXT is empty.

**Delivery:** Silent — no Telegram channel binding. CI Monitor has no channel; dream routine completes without notification.
