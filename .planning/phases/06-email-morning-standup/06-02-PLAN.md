Key implementation points:
- CLIENT_ID and CLIENT_SECRET from process.env (OPENCLAW_GMAIL_CLIENT_ID / OPENCLAW_GMAIL_CLIENT_SECRET)
- REDIRECT_URI = 'http://127.0.0.1:8080' (localhost loopback — OOB is deprecated)
- SCOPES = ['https://www.googleapis.com/auth/gmail.readonly', 'https://www.googleapis.com/auth/gmail.send', 'https://www.googleapis.com/auth/gmail.modify']
- generateAuthUrl({ access_type: 'offline', prompt: 'consent', scope: SCOPES })
  NOTE: prompt:'consent' is REQUIRED to receive refresh_token on every authorization
- http.createServer to listen on port 8080, capture ?code= from redirect
- await oauth2Client.getToken(code) → tokens.refresh_token
- Store via: security add-generic-password -s openclaw.gmail-triage-refresh-token -a echo.sys.bot@gmail.com -w TOKEN -U
  NOTE: -U flag = upsert (update if exists, create if absent)
- Print the browser URL clearly — user must open it manually OR script can try open(1) on macOS

- CRITICAL: Never log the refresh token value to console — only confirm storage
- Use `require('googleapis')` — the googleapis package must be installed in this scripts/ directory by Plan 06-01 before this script can run (it will be)

BEFORE running the script, complete these prerequisites:

1. Create a Google Cloud project (or use an existing one) for echo.sys.bot@gmail.com:
   - Go to console.cloud.google.com
   - Enable Gmail API: APIs & Services → Library → Gmail API → Enable
   - Create OAuth consent screen: APIs & Services → OAuth consent screen → External → fill in App name/email
   - Add test user: OAuth consent screen → Test users → + Add Users → echo.sys.bot@gmail.com

2. Create Desktop App OAuth2 credential:
   - APIs & Services → Credentials → + Create Credentials → OAuth client ID
   - Application type: Desktop app
   - Download the JSON or copy Client ID and Client Secret

3. Store credentials in Keychain:
   ```zsh
   security add-generic-password -s 'openclaw.gmail-client-id' -a echo.sys.bot@gmail.com -w 'YOUR_CLIENT_ID' -U
   security add-generic-password -s 'openclaw.gmail-client-secret' -a echo.sys.bot@gmail.com -w 'YOUR_CLIENT_SECRET' -U
   ```

4. Source the env vars (so the script can read them):
   ```zsh
   source ~/.openclaw/scripts/openclaw-secrets.sh
   ```

5. Run the authorization script:
   ```zsh
   cd ~/.openclaw/agents/email-triage/scripts
   node oauth2-setup.js
   ```
   - A browser window should open (or copy-paste the printed URL)
   - Sign in as echo.sys.bot@gmail.com and approve the requested Gmail scopes
   - The terminal should print "Refresh token stored in Keychain as openclaw.gmail-triage-refresh-token"

6. Verify the token was stored:
   ```zsh
   security find-generic-password -s 'openclaw.gmail-triage-refresh-token' -w >/dev/null && echo "TOKEN STORED" || echo "TOKEN MISSING"
   ```

7. Restart the gateway to propagate the token to the agent env:
   ```zsh
   zsh ~/agentic-setup/scripts/stow-deploy.sh
   launchctl kickstart -k gui/$(id -u)/ai.openclaw.gateway
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser OAuth2 consent → localhost:8080 | Authorization code arrives via HTTP redirect to local server |
| Authorization code → Google token endpoint | Code exchanged for refresh token over HTTPS |
| refresh_token value → macOS Keychain | Token written via security CLI; never touches filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | oauth2-setup.js refresh token value | mitigate | Script never logs refresh token value; only confirms storage; token goes directly from getToken() to security CLI via execSync |
| T-06-06 | Tampering | localhost:8080 redirect capture | accept | localhost loopback cannot be intercepted by remote hosts; standard Google-recommended approach for Installed App flow |
| T-06-07 | Information Disclosure | Client secret in env var | mitigate | OPENCLAW_GMAIL_CLIENT_SECRET read from Keychain via openclaw-secrets.sh; never appears in scripts or git history |
| T-06-SC | Tampering | npm install googleapis@172.0.0 | accept | Package legitimacy confirmed in RESEARCH.md — official Google org, approved |
</threat_model>
