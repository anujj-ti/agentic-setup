Step B — Create scripts/standup-brief.sh:
Create the file at scripts/standup-brief.sh with these exact characteristics:

Shebang and strict mode:
```zsh
#!/usr/bin/env zsh
set -euo pipefail
```

Source the json-response lib (relative path from script location):
```zsh
source "$(dirname "$0")/lib/json-response.sh"
```

Argument parsing: The script MUST accept `--repo OWNER/REPO` as a required argument. Parse with a while loop checking `$1`. If --repo is missing, call `json_err "Usage: standup-brief.sh --repo OWNER/REPO"`. Store repo in REPO variable.

Explicit binary paths (per CLAUDE.md nvm shadowing protection):
```zsh
GH=/opt/homebrew/bin/gh
JQ=/opt/homebrew/bin/jq
```

Verify gh is available: `[[ -x "$GH" ]] || json_err "gh not found at $GH — run: brew install gh"`.
Verify jq is available: `[[ -x "$JQ" ]] || json_err "jq not found at $JQ — run: brew install jq"`.

Data aggregation — three gh queries, all with `2>/dev/null` fallback to `[]` if query fails:

1. Merged PRs (last 24h): use `$GH pr list --state merged --search "merged:>$(date -u -v-24H '+%Y-%m-%dT%H:%M:%SZ')" --json number,title,mergedAt,mergedBy --limit 20 --repo "$REPO"`. Assign to MERGED_PRS. If the command fails, MERGED_PRS='[]'.

2. CI failures: `$GH run list --status failure --json name,conclusion,headBranch,url,createdAt --limit 10 --repo "$REPO"`. Assign to CI_FAILURES. Fallback to '[]'.

3. Stale PRs: `$GH pr list --state open --json number,title,updatedAt,reviewDecision,reviewRequests --limit 30 --repo "$REPO"` piped through `$JQ '[.[] | select((.reviewRequests | length) > 0 or .reviewDecision == "CHANGES_REQUESTED") | {number, title, updatedAt, reviewDecision}]'`. Assign to STALE_PRS. Fallback to '[]'.

Compose output JSON (use printf to avoid shell quoting issues):
```zsh
json_ok "{\"repo\":\"$REPO\",\"as_of\":\"$(date -u '+%Y-%m-%dT%H:%M:%SZ')\",\"merged_prs\":$MERGED_PRS,\"ci_failures\":$CI_FAILURES,\"stale_prs\":$STALE_PRS}"
```

Make the file executable: `chmod +x scripts/standup-brief.sh`

IMPORTANT: The script targets macOS only (CLAUDE.md constraint). Use `date -u -v-24H` (BSD date format). Never use GNU date flags.
