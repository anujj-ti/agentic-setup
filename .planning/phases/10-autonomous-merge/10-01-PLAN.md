It reads OPENCLAW_NOTION_TOKEN and OPENCLAW_NOTION_DECISIONS_DB_ID from process.env — if either is missing, write {"ok":false,"error":"missing env var"} to stderr and exit 1. Step 2 — Notion pre-log (MUST succeed before merge): capture output of node scripts/notion-log-decision.js with args: --action "merge PR #${PR_NUMBER}" --rationale "CI passing, quality review passed, autonomous merge authorized" --reversibility "reversible via devbot-revert-merge.sh: git revert <sha> + gh pr reopen" --evidence "statusCheckRollup: all SUCCESS on PR #${PR_NUMBER}". Capture the returned page ID into PAGE_ID. Check [[ -n "$PAGE_ID" ]] — if empty or the node script exited non-zero (set -e catches this), print {"ok":false,"error":"Notion pre-log failed — merge blocked per D-100","prNumber":"${PR_NUMBER}"} and exit 1.

Step 3 — Squash merge (per D-101): run /opt/homebrew/bin/gh pr merge "$PR_NUMBER" --squash --delete-branch. This is only reachable after PAGE_ID is confirmed. Per D-101, squash is used so the resulting commit is a plain (non-merge) commit — reverting it requires no -m flag.

Step 4 — Capture merge commit SHA (per D-102): run /opt/homebrew/bin/gh pr view "$PR_NUMBER" --json mergeCommit --jq '.mergeCommit.oid'. Store as MERGE_SHA. Run node scripts/notion-update-page.js --pageId "$PAGE_ID" --mergeCommitSha "$MERGE_SHA".
