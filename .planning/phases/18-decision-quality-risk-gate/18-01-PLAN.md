Append a "## Risk Scoring (RISK-01)" section to the existing SOUL.md — do NOT alter any existing content above. The new section must contain:

**Four Scoring Dimensions** (per D-502) with weights that sum to 100:
- Reversibility (40 pts): irreversible = 35-40, complex-revert = 20-30, simple-revert = 5-15, no-state-change = 0-5
- Blast radius (30 pts): multi-repo/multi-user = 25-30, single-repo = 15-20, single-file/single-issue = 5-10, read-only = 0
- External side effects (20 pts): email send / webhook / external API write = 15-20, GitHub mutation (merge/close/create) = 8-12, Notion write only = 3-5, read-only = 0
- Action recency (10 pts): first-ever occurrence of this action class = 8-10, seen before with clean history = 0-3

**Tier mapping** (per D-503):
- 0-30 = low (auto-proceed — no gate)
- 31-60 = medium (silent — no blocking gate; async notify deferred to v2.1)
- 61-100 = high (synchronous Telegram approval required before Notion write)

**Scoring instructions** for the LLM:
- Score each dimension using the ranges above; sum the four scores to get risk_score.
- Derive each dimension directly from the decision payload already under review — no external lookups.
- Reversibility maps from the existing `reversibility` field: "irreversible" → 35-40, "reversible" → 5-15.
- Blast radius: infer from the `decision` field; "merge PR" → single-repo (15-20); "create issue" → 5-10; "gh api ... --method GET" → 0.
- External side effects: infer from action verb; "gh pr merge", "gog gmail send" → 15-20; "gh issue create" → 8-12; "notion log" → 3-5.
- Action recency: use the rationale and evidence fields; if they describe a novel operation, score 8-10; if they reference prior successful runs, score 0-3.
